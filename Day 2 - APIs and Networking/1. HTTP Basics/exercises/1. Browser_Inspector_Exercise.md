## Exercise 3: curl POST, PUT, PATCH, DELETE

### Task 1: POST - Create a New Resource

**Bash:**
```bash
curl -i -X POST https://jsonplaceholder.typicode.com/posts \
  -H "Content-Type: application/json" \
  -d '{"title": "Hello World", "body": "My first POST", "userId": 1}'
```

**PowerShell:**
```powershell
$body = @{
    title  = "Hello World"
    body   = "My first POST"
    userId = 1
} | ConvertTo-Json

Invoke-WebRequest -Uri "https://jsonplaceholder.typicode.com/posts" `
  -Method POST `
  -ContentType "application/json" `
  -Body $body
```

Questions to answer:
- What status code did you get? (hint: it's not `200`)
- What `id` was assigned to the new post?
- What does the `Location` header tell you?

### Task 2: PUT - Full Update of a Resource

**Bash:**
```bash
curl -i -X PUT https://jsonplaceholder.typicode.com/posts/1 \
  -H "Content-Type: application/json" \
  -d '{"id": 1, "title": "Updated Title", "body": "Completely replaced body", "userId": 1}'
```

**PowerShell:**
```powershell
$body = @{
    id     = 1
    title  = "Updated Title"
    body   = "Completely replaced body"
    userId = 1
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://jsonplaceholder.typicode.com/posts/1" `
  -Method PUT `
  -ContentType "application/json" `
  -Body $body
```

Questions to answer:
- What is the difference between PUT and PATCH?
- What happens if you omit the `body` field in a PUT request?

### Task 3: PATCH - Partial Update

**Bash:**
```bash
curl -i -X PATCH https://jsonplaceholder.typicode.com/posts/1 \
  -H "Content-Type: application/json" \
  -d '{"title": "Only the title changed"}'
```

**PowerShell:**
```powershell

Invoke-RestMethod -Uri "https://jsonplaceholder.typicode.com/posts/1" `
  -Method PATCH `
  -ContentType "application/json" `
  -Body $body
```

### Task 4: DELETE - Remove a Resource

**PowerShell:**

## Exercise 4: Headers, Auth, and Request Inspection with httpbin

> `httpbin.org` is a purpose-built HTTP mirror тАФ it echoes back exactly what you sent. Perfect for learning.

### Task 1: Send Custom Headers

**Bash:**
```bash
curl https://httpbin.org/headers \
  -H "X-My-Name: YourName" \
  -H "X-Course: BlitzLearning"
```

**PowerShell:**
```powershell
$headers = @{
    "X-My-Name" = "YourName"
    "X-Course"  = "BlitzLearning"
}
Invoke-RestMethod -Uri "https://httpbin.org/headers" -Headers $headers
```

Questions to answer:
- Can you see your custom headers echoed back in the response?
- What other headers did curl/PowerShell add automatically?

### Task 2: Send Query Parameters and See Them Echoed

**Bash:**
```bash
curl "https://httpbin.org/get?search=cats&page=2&limit=10"
```

**PowerShell:**
```powershell
Invoke-RestMethod -Uri "https://httpbin.org/get?search=cats&page=2&limit=10"
```

Questions to answer:
- Where do the query parameters appear in the response?
- What is your public IP address? (hint: check the `origin` field)

### Task 3: Basic Authentication

**Bash:**
```bash
# This should succeed (credentials match the URL pattern)
curl -i -u student:password123 https://httpbin.org/basic-auth/student/password123
```

```bash
# This should fail - wrong password
curl -i -u student:wrongpassword https://httpbin.org/basic-auth/student/password123
```

**PowerShell:**
```powershell
# Successful auth
$creds = [Convert]::ToBase64String([Text.Encoding]::ASCII.GetBytes("student:password123"))
Invoke-WebRequest -Uri "https://httpbin.org/basic-auth/student/password123" `
  -Headers @{ Authorization = "Basic $creds" }
```

Questions to answer:
- What status code do you get for the failed attempt?
- Look at the `Authorization` header in verbose mode тАФ how is the password encoded?

### Task 4: POST with Form Data vs JSON

**Bash:**
```bash
# Form data (application/x-www-form-urlencoded)
curl -X POST https://httpbin.org/post \
  -d "username=rens&course=blitz"

# JSON body (application/json)
curl -X POST https://httpbin.org/post \
  -H "Content-Type: application/json" \
  -d '{"username": "rens", "course": "blitz"}'
```

**PowerShell:**
```powershell
# Form data
Invoke-RestMethod -Uri "https://httpbin.org/post" -Method POST -Body "username=rens&course=blitz"

# JSON body
$body = @{ username = "rens"; course = "blitz" } | ConvertTo-Json
Invoke-RestMethod -Uri "https://httpbin.org/post" -Method POST -ContentType "application/json" -Body $body
```

Questions to answer:
- In the response, where does form data appear vs JSON data?
- What `Content-Type` header did each request send?

## Exercise 5: HTTP Status Codes in Action

> Use `httpbin.org/status/{code}` to trigger any status code on demand.

### Task 1: Trigger Specific Status Codes

**Bash:**
```bash
curl -i https://httpbin.org/status/200
curl -i https://httpbin.org/status/201
curl -i https://httpbin.org/status/400
curl -i https://httpbin.org/status/401
curl -i https://httpbin.org/status/403
curl -i https://httpbin.org/status/404
curl -i https://httpbin.org/status/500
```

**PowerShell:**
```powershell
# Note: Invoke-WebRequest throws on 4xx/5xx by default, use -SkipHttpErrorCheck
foreach ($code in @(200, 201, 400, 401, 403, 404, 500)) {
    $r = Invoke-WebRequest -Uri "https://httpbin.org/status/$code" -SkipHttpErrorCheck
    Write-Host "Status $code -> $($r.StatusCode) $($r.StatusDescription)"
}
```

### Task 2: Follow a Redirect

**Bash:**
```bash
# Without -L: see the redirect response
curl -i https://httpbin.org/redirect/3

# With -L: follow all redirects automatically
curl -iL https://httpbin.org/redirect/3
```

**PowerShell:**
```powershell
# PowerShell follows redirects by default - disable to see the raw 302
Invoke-WebRequest -Uri "https://httpbin.org/redirect/3" -MaximumRedirection 0 -SkipHttpErrorCheck
```

Questions to answer:
- What status code signals a redirect?
- What header tells the client where to go next?

## Challenge Tasks

### Challenge 1: Chain Two Requests Together
Fetch a post from JSONPlaceholder, extract the `userId`, then fetch that user's profile тАФ all in one pipeline.

**Bash:**
```bash
USER_ID=$(curl -s https://jsonplaceholder.typicode.com/posts/5 | python3 -c "import sys,json; print(json.load(sys.stdin)['userId'])")
curl -s "https://jsonplaceholder.typicode.com/users/$USER_ID"
```

**PowerShell:**
```powershell
$post   = Invoke-RestMethod -Uri "https://jsonplaceholder.typicode.com/posts/5"
$userId = $post.userId
Invoke-RestMethod -Uri "https://jsonplaceholder.typicode.com/users/$userId"
```

### Challenge 2: Measure Response Time

**Bash:**
```bash
# Built-in timing with curl write-out
curl -o /dev/null -s -w "DNS: %{time_namelookup}s | Connect: %{time_connect}s | Total: %{time_total}s\n" \
  https://jsonplaceholder.typicode.com/posts

# Compare a large vs small payload
curl -o /dev/null -s -w "Total: %{time_total}s | Size: %{size_download} bytes\n" \
  https://jsonplaceholder.typicode.com/posts

curl -o /dev/null -s -w "Total: %{time_total}s | Size: %{size_download} bytes\n" \
  https://jsonplaceholder.typicode.com/posts/1
```

**PowerShell:**
```powershell
$start    = Get-Date
$response = Invoke-WebRequest -Uri "https://jsonplaceholder.typicode.com/posts"
$elapsed  = (Get-Date) - $start
Write-Host "Time: $($elapsed.TotalMilliseconds)ms | Size: $($response.RawContentLength) bytes"
```

### Challenge 3: Security Check
Use httpbin to inspect what your requests reveal about you.

**Bash:**
```bash
# What does the server see about your request?
curl https://httpbin.org/get

# Does HTTPS actually encrypt your headers?
curl -v https://httpbin.org/get 2>&1 | grep -E "SSL|TLS|subject|issuer"

# What user-agent are you sending?
curl https://httpbin.org/user-agent

# Override your user-agent
curl https://httpbin.org/user-agent -A "MyCustomBrowser/1.0"
```

**PowerShell:**
```powershell
# Check your user-agent
Invoke-RestMethod -Uri "https://httpbin.org/user-agent"

# Override user-agent
Invoke-RestMethod -Uri "https://httpbin.org/user-agent" `
  -Headers @{ "User-Agent" = "MyCustomBrowser/1.0" }
```

## curl Quick Reference

| Flag | Purpose | Example |
|------|---------|---------|
| `-i` | Include response headers | `curl -i URL` |
| `-v` | Verbose (full conversation) | `curl -v URL` |
| `-I` | HEAD request only | `curl -I URL` |
| `-X` | Set HTTP method | `curl -X POST URL` |
| `-H` | Add a header | `curl -H "Key: Value" URL` |
| `-d` | Request body | `curl -d '{"a":1}' URL` |
| `-u` | Basic auth | `curl -u user:pass URL` |
| `-L` | Follow redirects | `curl -L URL` |
| `-s` | Silent (no progress) | `curl -s URL` |
| `-o` | Save output to file | `curl -o file.json URL` |
| `-w` | Print timing/stats | `curl -w "%{time_total}" URL` |
